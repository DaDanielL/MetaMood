# Codex AI Layer

This directory contains reusable Codex workflows for turning an idea into implemented, reviewed work. It is intentionally app-neutral: no demo app, framework scaffold, or product code lives here.

## Artifact Directories

| Artifact | Path | Purpose |
|----------|------|---------|
| PRDs | `.agents/PRDs/` | Product requirements generated by `prd-interactive` or `create-prd` |
| Story manifests | `.agents/stories/` | GitHub issue-ready story breakdowns from `create-stories` |
| Plans | `.agents/plans/` | Implementation plans generated by `plan` |
| Reports | `.agents/reports/` | Implementation reports generated by `implement` |
| Reviews | `.agents/reviews/` | Code review reports generated by `review` |
| Skills | `.agents/skills/` | Workflow definitions |

## Workflow Order

1. `rules-interactive` or `create-rules`
2. `prd-interactive` or `create-prd`
3. `create-stories`
4. `prime`
5. `plan`
6. `implement`
7. `validate`
8. `review` / `security-review`

## Workflows

| Workflow | Use When | Output |
|----------|----------|--------|
| `rules-interactive` | Greenfield project with a PRD but unknown stack, architecture, tools, and folder structure | Root `AGENTS.md` |
| `create-rules` | Existing project needs Codex rules extracted from the codebase | Root `AGENTS.md` |
| `prd-interactive` | Product idea needs guided discovery before writing a PRD | `.agents/PRDs/{name}.prd.md` |
| `create-prd` | Requirements already exist in the conversation and should be formalized | `.agents/PRDs/{name}.prd.md` |
| `create-stories` | PRD should be broken into small, GitHub issue-ready stories | `.agents/stories/{name}.stories.md`; optionally GitHub Issues via `gh issue create` |
| `prime` | Codex needs project context before planning or implementation | Context summary |
| `prime-client` | Frontend-specific context is needed | Frontend context summary |
| `prime-server` | Backend-specific context is needed | Backend context summary |
| `prime-components` | UI component conventions are needed | Component pattern summary |
| `prime-endpoint` | API, webhook, CLI, or external contract patterns are needed | Contract pattern summary |
| `plan` | A PRD, story, issue, or feature needs an implementation plan | `.agents/plans/{name}.plan.md` |
| `implement` | A plan should be executed with validation loops | Code changes plus `.agents/reports/{name}-report.md` |
| `validate` | Current work needs project checks run and summarized | Validation report in chat |
| `review` | A PR, diff, file, or folder needs code review | `.agents/reviews/{scope}-review.md` |
| `security-review` | Changes need security-focused review | Security findings in chat or review notes |
| `install` | A real project needs dependency setup and local verification | Setup summary |

## GitHub Issues and PRs

This layer uses GitHub Issues for issue tracking.

Use GitHub CLI where useful:

```bash
gh issue view {number}
gh issue create
gh pr create
gh pr view {number}
```

Story manifests keep local `STORY-###` IDs even when GitHub Issues are created. This preserves traceability from PRD to story to issue to plan.

## AGENTS.md

`AGENTS.md` is the project rules file Codex should read first. Generate it with:

- `rules-interactive` for greenfield projects where stack and structure are undecided.
- `create-rules` for existing projects where the codebase already reveals conventions.

Use `.agents/AGENTS-template.md` as the base template.

## Recommended Greenfield Flow

1. Run `prd-interactive` to shape the product idea into `.agents/PRDs/{name}.prd.md`.
2. Run `rules-interactive .agents/PRDs/{name}.prd.md` to decide stack, architecture, tools, tests, folder structure, and key files, then generate `AGENTS.md`.
3. Run `create-stories .agents/PRDs/{name}.prd.md` to create a story manifest.
4. Optionally create GitHub Issues with `gh issue create`.
5. Run `prime`, then `plan` for the first story or issue.
6. Run `implement`, then `validate`.
7. Run `review` and `security-review` before merging.

## Template Rule

This repository is only the reusable AI-agent-layer template. Do not create a demo application inside it.
